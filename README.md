 HEAD
(Добавлен README)


`process-tools` — репозиторий проектов, шаблонов и ресурсов для автоматизации личных процессов.

---

## Назначение

- создание и сопровождение заметок (daily / weekly / monthly)
- инициализация и обслуживание сайтов (блоги, документация, лендинги)
- единый CLI для типовых операций
- хранение шаблонов и правил процесса
- управление внешними проектами из одного места

---



## Структура репозитория

```text
process-tools/
├─ apps/        # Проекты целиком (код + точки запуска)
├─ templates/   # Шаблоны (чистые файлы для копирования/генерации)
├─ resources/   # Библиотека (prompts/assets/datasets/links)
├─ meta/        # Правила и навигация по репозиторию
├─ dev/         # Локальная среда (venv/cache/logs/tmp) — НЕ в git
└─ README.md
````

---

## Проекты

### `apps/`

Здесь лежат **проекты целиком**. Каждый проект — отдельная папка со своим README и точкой запуска.

Пример:

```text
apps/
├─ telegram-bot/
│  ├─ README.md
│  ├─ run.py
│  └─ src/
└─ site-tools/
   ├─ README.md
   └─ src/
```

---

## Шаблоны

### `templates/`

Содержит **чистые шаблоны**, которые копируются или используются генераторами:

```text
templates/
├─ notes/
├─ sites/
└─ storage/
```

* Markdown
* HTML
* конфиги генераторов
* заготовки структур

---

## Ресурсы

### `resources/`

Библиотека переиспользуемых материалов:

```text
resources/
├─ prompts/     # готовые промты и сценарии
├─ assets/      # изображения/иконки/шрифты/заготовки
└─ datasets/    # датасеты и источники
```

Рекомендация для датасетов:

```text
resources/datasets/
├─ sample/      # маленькие примеры (можно коммитить)
└─ external/    # ссылки/инструкции/скрипты загрузки
```

---

## Правила репозитория

### `meta/`

Здесь лежат **правила и соглашения**, чтобы структура не расползалась:

* куда класть новый проект
* где хранить промты
* что считать шаблоном, а что ресурсом
* стандарты именования (если нужны)

---

## Dev-среда

### `dev/`

Локальные артефакты, которые можно удалить и пересоздать:

```text
dev/
├─ .venv/   # виртуальное окружение
├─ cache/
├─ logs/
└─ tmp/
```

`dev/` не коммитится.

---



## Статус проекта

Проект развивается итеративно.
Структура может расширяться, но базовые принципы сохраняются.

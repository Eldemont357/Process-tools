
# templates

`templates/` — это **шаблоны**, которые копируются/генерируются в проектах из `apps/`.
Здесь нет “живых данных” и нет логики запуска.

## Что здесь хранить
- шаблоны файлов: `.md`, `.txt`, `.yaml`, `.json`, `.html`
- шаблоны структур папок (каркасы)
- “болванки” сообщений, страниц, конфигов

## Что здесь НЕ хранить
- реальные данные (отчёты, фотки, выгрузки)
- секреты/токены
- виртуальные окружения
- исполняемый код (это в `apps/`)

## Рекомендуемая структура

```text
templates/
├─ notes/        # заметки (daily/weekly/...)
├─ sites/        # каркасы сайтов
├─ messages/     # шаблоны сообщений (боты/уведомления)
└─ storage/      # шаблоны структур хранения (папки)
````

## Формат шаблона (правило)

Каждый шаблон — отдельная папка:

```text
templates/<group>/<template-name>/
├─ meta.yaml         # описание и параметры (опционально, но желательно)
└─ template/         # содержимое шаблона (файлы/папки)
```

Если шаблон — один файл, можно проще:

```text
templates/<group>/<template-name>/
├─ meta.yaml
└─ template.md
```

## Как использовать

Шаблоны не запускаются напрямую.
Их используют проекты из `apps/`:

* копируют файлы/папки
* подставляют переменные (если ты это реализуешь)
* создают результат в нужном месте

## Принцип

* `templates/` = форма (как выглядит результат)
* `apps/` / `packages/` = логика (как получить результат)
